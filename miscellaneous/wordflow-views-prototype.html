Wordflow views

把周期笔记里的写作轨迹，渲染成可复盘的 Obsidian 视图

首批原型只使用现有记录器能解析的数据：周期笔记 metadata、table、bullet list，以及被修改笔记的 metadata、tags 和 folder。

38h 42m专注总时长
21,840编辑字数
128编辑次数
34活跃笔记
语法草案 短关键词，按自然阅读顺序组合
```wordflow
SUMMARY
RECORDER "Daily Tracker"
PAST 7 DAYS
VALUE readEditTime
FILTER tags: (#writing AND #project) AND (-#archive OR -#draft)
FILTER folder: "Notes/Projects"
TITLE "本周写作复盘"
```
视图HEATMAP, SUMMARY, DONUT, BAR, STACKED, TIMELINE
时间TODAY, WEEK, MONTH, PAST 12 WEEKS, FROM ... TO ...
过滤tags: 支持 AND/OR/-/括号；folder: 支持路径前缀

10 个扩展视图草案

这些先作为第二批候选：有些适合马上做，有些需要等解析和聚合能力更稳定后再做。

IDEAS 10 view types prototype only

1. NOTE TREND 单文件趋势线

目的：回答“这个文件本周每天存量字数/历史编辑次数怎么变”。无数据的日期沿用上一条值，曲线保持平，不回落到 0。

MonTue no dataWedThu flatFriSat flatSun

美观关键：用阶梯/平台表达“存量”，虚线平台表示该日没有新记录但值被 carry forward。

NOTE TREND FILE "Projects/Wordflow Tracker.md" THIS WEEK VALUE docWords, history.editedTimes FILL previous

2. NOTE FISHBONE 浏览鱼骨线

目的：回答“我本周每天浏览/编辑过哪些笔记或 tag group”。一天是一条主鱼骨，多条子骨代表当天的笔记或 tag group。

tag groupnote
Mon Tue Wed Thu Fri Sat Sun Plugin2h12m API note3h05m Reading1h48m Draft4h20m Roadmap2h40m Inbox56m Wordflow5h18m Settings1h24m Spec2h03m

美观关键：主骨是日期，子骨数量表示当天覆盖面，子骨长度表示 `readEditTime`，颜色来自笔记/tag group 配置。

FISHBONE RECORDER "Daily Tracker" THIS WEEK GROUP BY day SPLIT BY tag LABEL noteTitle

6. STREAM 堆叠曲线流图

目的：回答“周一到周日不同笔记的专注时长如何叠加”。适合看注意力迁移。

美观关键：柔和流线比普通堆叠柱更能表现“注意力流动”。

STREAM THIS WEEK GROUP BY note VALUE readEditTime LIMIT 6

7. CALENDAR LANES 日历泳道

目的：把专注数据贴到日历上下文里，看规律和异常。适合月视图、季度视图。

MTWTFSS AM PM Night

美观关键：像迷你日历，不只是图表；时间段可由 `lastModifiedTime` 或记录时间推断。

CALENDAR LANES THIS MONTH VALUE readEditTime SPLIT BY timeOfDay

8. TAG FLOW 标签流向图

目的：回答“时间从哪些 folder 流向哪些 tag group”。适合发现项目结构和标签体系是否一致。

美观关键：带宽表示 `readEditTime` 或 `editedWords`，适合“文件夹 -> 标签”的双维分析。

FLOW PAST 90 DAYS FROM folder TO tag VALUE readEditTime

9. BURNUP / BURNDOWN 目标进度线

目的：服务目标管理，比如“本月写 50,000 字还差多少”。如果用户设目标，这个非常有用。

美观关键：目标线、实际累计线、剩余线同屏；没有目标时不建议默认展示。

BURNUP THIS MONTH VALUE addedWords TARGET 50000

10. CFD 累积流图：笔记变体

目的：把 cumulative flow 的“累积结构”换成笔记维度，看最近 7 天注意力是否长期集中在少数笔记。

MonTueWedThuFriSatSun

美观关键：每个颜色是一篇笔记或 Top N 笔记，横轴明确写周一到周日；其他笔记合并为“Other”。

CFD THIS WEEK GROUP BY note VALUE readEditTime LIMIT 5

10b. CFD 累积流图：tag group 变体

目的：同一个结构换成 tag group，回答“最近 7 天注意力主要堆在哪些主题”。

D-6D-5D-4D-3D-2D-1Today

美观关键：颜色严格使用 tag group 配置色；适合最近 7 天或最近 12 周，不适合当天详情。

CFD PAST 7 DAYS GROUP BY tag VALUE editedTimes LIMIT 6

首批视图取舍

只保留目的明确、能从现有数据直接产出的视图；饼图先作为 DONUT 的 `STYLE pie` 变体，不单独占一个视图类型。

多数用户会用

SUMMARY/HEATMAP/BAR/TIMELINE 都服务于复盘：我哪天写得多、主要写了什么、最近改了哪些笔记。

这些目的清晰，不依赖额外数据建模。

适合的数据

周期 metadata 适合热力图；table/list 适合单笔记聚合、标签和文件夹过滤；被修改笔记 metadata 适合历史总量。

`statBar` 最适合 STACKED，不建议只压成单个数字。

视觉方向

保持 Obsidian 内嵌感：浅边框、低阴影、紧凑密度；用多色表达字段含义，而不是用大面积装饰。

热力图和时间线负责“眼前一亮”，看板负责“马上有用”。
